# meeting memory agent 的记忆存储
记忆管理 agent 需要处理两类记忆
- 每局游戏的信息
- 鹅鸭杀游戏的规则玩法

## 短期记忆
每局游戏的信息，由于对于其他局游戏的影响小，不必跨局（跨线程）记录，采用短期记忆。

两种实现方式：
- `InMemorySaver`，完全存储于内存中。由于多模态输入 agent 的输出为纯文本格式，占用内存或较小，能尝试使用。
- `PostgreSQL`，体量相对较大，但对于windows系统操作较为简便。
- `RedisSaver`，与数据库连接进行存储。需要提前在电脑中下载`Docker`，并运行`RedisStack`，[步骤网址](https://redis.io/docs/latest/operate/oss_and_stack/install/install-stack/docker/)。

> 为何选择 `Redis`？
> 1. PostgreSQL 过于庞大
> 2. SQLite 轻量化，python 原生自带，但不支持 JSON 格式
> 3. Redis 也需要选用 Redis Stack 包含 RedisJSON 和 RedisSearch，才可支持上述功能

## python tips
```python
class ShortMemoryStore:
    def __init__(self):
        self.DB_URI = short_memory_conf['DB_URL']
    
    async def get_checkpointer(self):
        return await AsyncSqliteSaver.from_conn_string(get_abs_path(self.DB_URI))
```
为什么 return 的不是 `BaseCheckpointSaver`？
当你执行 `AsyncSqliteSaver.from_conn_string(...)` 时，它返回的类型确实不是 `BaseCheckpointSaver`，而是 `_AsyncGeneratorContextManager`。
原因：工厂模式与装饰器
`from_conn_string` 在内部被 `@asynccontextmanager` 装饰了。这意味着：

它变成了一个生成器工厂。

它返回的是一个包装器（Wrapper）。这个包装器的作用是告诉你：“我可以给你一个 BaseCheckpointSaver 实例，但前提是你得用 `async with` 邀请我出来。”


```python
    @asynccontextmanager
    async def get_saver(self):
        """返回带自定义 serde 的 checkpointer，支持 IngestionOutput/MemorySummary 的序列化"""
        serde = JsonPlusSerializer(allowed_msgpack_modules=ALLOWED_MSGPACK_MODULES)
        
        # 建立物理连接。使用 aiosqlite（异步版 SQLite 驱动）打开数据库文件。
        async with aiosqlite.connect(get_abs_path(self.DB_URI)) as conn: 
            yield AsyncSqliteSaver(conn, serde=serde)
```
|疑问,形象理解|装饰器,厨房入场券（允许你进出）|
|-|-|
|yield 之前|厨师备菜（建立连接）|
|yield 本身|把菜递出窗口（把 saver 给外部）|
|外部使用期间|厨师在窗口等着（函数暂停）|
|yield 之后|厨师收摊关门（关闭连接）|

- 装饰器 `@asynccontextmanager`：它让你的函数具备了处理 `async with` 的能力。

```python
@asynccontextmanager
async def get_saver(self):
    print("--- 1. 准备连接数据库 ---")
    async with aiosqlite.connect(...) as conn:
        print("--- 2. 连接已建立，交给外部使用 ---")
        yield AsyncSqliteSaver(conn, serde=serde)
        print("--- 4. 外部用完了，我准备关门了 ---")
    print("--- 5. 数据库连接彻底断开 ---")

# 外部测试
```
